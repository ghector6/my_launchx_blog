Now what ‚ùî

Our test confirms that we can add a single todo but now we need a way to see if it is only going to be one todo. we can do this like this:

  1. With the dev tools we need to inspect where is our todo added.
  2. Now that we know where it is and the type of element, a li this time, we make an assertion that only one li element exists in our app todo üëì:
describe("React TodoMVC", () => {
  it("adds a single todo", () => {
    cy.visit("http://localhost:8888")
    cy.get(".new-todo").type("Learn Cypress{enter}")
    cy.get(".todo-list li").should("have.length", 1)
  })
})
  • If we add another todo to the test it is goig to break üñçÔ∏è.
describe("React TodoMVC", () => {
  it("adds a single todo", () => {
    cy.visit("http://localhost:8888")
    cy.get(".new-todo").type("Learn Cypress{enter}")
    cy.get(".new-todo").type("Learn Java{enter}")
    cy.get(".todo-list li").should("have.length", 1)
  })
})
  • We can simplify this statement like this and change the length in order to make it pass. ü•á
describe("React TodoMVC", () => {
  it("adds a single todo", () => {
    cy.visit("http://localhost:8888")
    cy.get(".new-todo").type("Learn Cypress{enter}").type("Learn JavaScript{enter}")
    cy.get(".todo-list li").should("have.length", 2)
  })
})
